## P1-12 — Doctor inaccuracies & delete lineage gap

**Problems.**

1. The FTS5 check was gated on the *embedding server's* health flag. A down
   embedding server skipped the probe and printed `fts5: ✓ available`
   unchecked, hiding genuine FTS corruption behind a false positive.
2. The `/health` and `/v1/models` probes are llama.cpp/ollama conventions. For
   `claude`/`openai` providers they hit the wrong path on a hosted API (or an
   empty `BaseURL`), reporting misleading status.
3. `tbuk delete` removed the document and its chunks but left the
   extracted-text cache file (`extractedDir/<sha>.txt`) behind forever — a
   lineage-cleanup gap.

**Fix.**

- `runDoctor` now tracks database health in its own `dbOK` variable and gates
  the FTS5 check on it, independent of any HTTP server reachability.
- Added `isHostedProvider(provider)` (`claude`/`openai`). Both the LLM and
  embedding sections skip HTTP probes for hosted providers and print
  `hosted API — not probed; set ANTHROPIC_API_KEY/OPENAI_API_KEY`. The model
  probe (`/v1/models`) is likewise skipped for hosted LLM providers, falling
  back to the configured model name.
- `runDoctor` was refactored to write to an `io.Writer`; exported
  `RunDoctorTo(w, client, cfg, cfgPath)` alongside the stdout-writing
  `RunDoctor`. `makeDoctorRunner` writes to `cmd.OutOrStdout()`.
- `RunDelete` gained an `extractedDir` parameter and removes
  `extractedDir/<sha>.txt` after deleting the document (best-effort — a missing
  file is not an error; a real removal error is surfaced). The `delete` command
  passes `cfg.Preprocess.OutputDir`.

**Tests.**

- `TestRunDoctorTo_fts5CheckedWhenEmbedderDown`: a DB with `chunks_fts` dropped
  and an unreachable embedder reports `fts5: ✗` where the old code printed a
  bogus `✓`.
- `TestRunDoctorTo_hostedProviderNotProbed`: `claude`/`openai` providers yield
  zero HTTP hits against a counting server and the report says `not probed`.
- `TestRunDelete_removesExtractedCacheFile`: delete removes the cache file.
- `TestRunDelete_missingCacheFileIsNotAnError`: a missing cache file leaves
  delete successful.

**Docs.** `README.md` (delete + doctor troubleshooting rows) and
`docs/initial-context.md` (doctor + delete CLI notes) updated to match.
