# Subplan 11 — P0-3: automatic metadata + `tbuk meta` commands

**Status: Implemented.** Extracted from `docs/plans/11-poc-hardening.md`
(PR 3 of the phasing table). The remaining phases stay in the active plan.

## Problem

`Ingester` received a `MetadataRepo` but never called it. No code path wrote
metadata rows, so `tbuk find key=value` (a headline POC feature —
`tbuk find tag=design`, `tbuk find filename=README.md`) could never match.

## Fix (delivered)

Two parts:

1. **Automatic metadata on ingest.** `IngestFile` now writes four automatic
   keys per document via `MetadataRepo.Set` (upsert):
   - `filename` — `filepath.Base(path)`
   - `extension` — extension lowercased, leading dot stripped (`md`, not `.md`)
   - `mime` — `preprocess.DetectMIME(path)`
   - `dir` — `filepath.Dir(path)`

   Keys refresh on every ingest; because `Set` upserts by key, user-set keys
   are left intact across re-ingest. This makes `tbuk find filename=README.md`
   and `tbuk find extension=md` work after a plain `tbuk ingest`.

2. **`tbuk meta` commands.** New `meta` command with two subcommands:
   - `tbuk meta set <path> <key=value>...` — attach one or more pairs to the
     document at `path`.
   - `tbuk meta list <path>` — print all metadata for the document, ordered
     by key.

   Both resolve the document by path (`DocumentRepo.GetByPath`) and surface a
   clear "document not found" error otherwise. Backed by a new
   `MetadataRepo.List(ctx, docID) ([]Metadata, error)` storage method.

## Tests (written first, TDD)

- `storage`: `List` returns entries ordered by key; empty document; closed-DB
  error path.
- `ingest`: ingest writes the four automatic keys; re-ingest refreshes the
  automatic keys while preserving a user-set `tag`.
- `cli`: `RunMetaSet`/`RunMetaList` round-trip; invalid `key=value` pair;
  document-not-found for set and list; empty-metadata message; missing-arg
  validation on the cobra commands.

## Notes / scope boundaries

- Path is used exactly as typed (not normalized). Path normalization is a
  separate later phase (P1-11, PR 6).
- No doctor changes: metadata does not affect any doctor probe (doctor fixes
  are P1-12, PR 8).
- Docs updated: `README.md` (command list + Metadata section) and
  `docs/initial-context.md` (ingest pipeline, command list, cli package note).
