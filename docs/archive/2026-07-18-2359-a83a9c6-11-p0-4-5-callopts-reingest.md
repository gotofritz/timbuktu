# Subplan 11 — P0-4 & P0-5: manifest CallOptions + transactional re-ingest

**Status: Implemented.** Extracted from `docs/plans/11-poc-hardening.md`
(PR 4 of the phasing table). The remaining phases stay in the active plan.

## P0-4 — Template manifest options never reached the LLM

### Problem

`cli/ask.go` computed a manifest `model` override into a local `llmCfg` and
then discarded it, and `RunAsk` called `chat(ctx, messages)` with no
`CallOptions`. Manifest `model`, `temperature: 0.2`, and `max_tokens: 2048`
were all silently ignored, breaking the "Template Capabilities" requirement.
`CallOptions.Temperature` was a plain `float64` that providers treated as
"unset" when `0`, so an explicit `temperature: 0` was not expressible.

### Fix (delivered)

- `CallOptions.Temperature` and `Manifest.Temperature` are now `*float64`.
  `nil` means "unset — use the provider default"; a non-nil pointer (including
  `0`) is sent. Claude/OpenAI providers omit `temperature` from the request
  body entirely when it is nil.
- `RunAsk` builds `llm.CallOptions{Model: manifest.Model, Temperature:
  manifest.Temperature, MaxTokens: manifest.MaxTokens}` and passes it into the
  chat call.
- The dead `llmCfg` block in `newAskCmd` is removed.
- `tbuk template show` prints `temperature: (default)` when unset.

## P0-5 — Re-ingest destroyed the index on embedding failure

### Problem

`IngestFile` called `chunks.DeleteByDocument` *before* extraction and
embedding. If embedding failed (provider down, 429), the old chunks were
already gone — the document's index was destroyed rather than left stale.
Subplan 06 required each file's chunks to be committed together or not at all.

### Fix (delivered)

- Removed the early `DeleteByDocument`.
- Added `ChunkRepo.ReplaceForDocument(ctx, docID, chunks)` which does
  `DELETE FROM chunks WHERE document_id=?` + bulk-insert inside a single
  transaction (rolls back on any error).
- `IngestFile` now extracts → chunks → embeds first, upserts the document row,
  then calls `ReplaceForDocument`. A failed re-ingest leaves the previous
  chunks intact and searchable.

## Tests (written first, TDD)

- `storage`: `ReplaceForDocument` replaces chunks and sets IDs; empty set
  clears existing chunks; closed-DB error path.
- `ingest`: `TestIngester_reingestEmbedErrorPreservesOldChunks` — a
  toggle embedder succeeds on first ingest then fails on the forced re-ingest;
  asserts the chunk count is unchanged (proved the bug first: `before=6
  after=0`).
- `cli`: `RunAsk` forwards manifest `CallOptions` (model, temperature 0.2,
  max_tokens 2048); explicit `temperature: 0` is forwarded as a non-nil `*0`.
- `llm`: Claude call sends `temperature` when set; omits it entirely when
  unset (`TestClaudeProvider_temperatureUnsetOmitted`).
