## 14. Dead and vestigial code — done

- `chunking.splitSentences` (returned `[]string{text}`, purely a non-empty
  guard) deleted; `Split` now inlines a `strings.TrimSpace(text) == ""`
  check. Doc comments in README / `initial-context.md` no longer describe a
  "greedy sentence accumulation" that never existed — the boundary search
  (`findBoundary`) does the work.
- `search/metadata.go`: removed the unused `alias` + `_ = alias`; JOIN
  aliases are now `fmt.Sprintf("m%d", i)`. The old `rune('0'+i)` produced the
  invalid token `:` for the 11th filter onward (`SQL logic error:
  unrecognized token: ":"`) — fixed and covered by a 12-filter test.
- Empty `internal/metadata` stub package deleted (it had no test files and
  dragged a "no test files" line through every coverage run). Re-adding a
  package later is cheap.
- `cli/ask.go` dead `llmCfg` block was already removed in PR 4.

## 15. Per-package coverage gate — done

AGENTS.md demands ≥ 85% per package; `cli` (82.7%) and `preprocess`
(82.2%) were below it and `make check-ci` only gated the total. Both raised
above 85% (cli 85.8%, preprocess 86.7%) with tests for the stats size query,
directory/file permissions, the `retrieval.max_tokens` trim, the `meta` /
`find` command paths, and preprocess error branches. `check-ci` now enforces
per-package coverage (fails on any package < 85% or with no test files) in
addition to the total gate.

## 16. Misc — done

- **No global mutable state**: `cli` no longer keeps `cfg` / `cfgFile` in
  package-level vars. Config is loaded in the root `PersistentPreRunE` and
  threaded through the cobra command context (`configFrom(cmd)` /
  `configPathFrom(cmd)`).
- **`stats` size**: `COALESCE(SUM(LENGTH(c.text)), 0)` replaces
  `LENGTH(GROUP_CONCAT(c.text))` — exact byte sum, no separator-comma
  overcount and no `group_concat` length-limit risk.
- **`retrieval.max_tokens`**: now enforced. `RunAsk` trims retrieved chunks
  to the manifest's approximate token budget before rendering (a
  non-positive budget disables trimming; at least one chunk is always kept).
- **Permissions hardening**: `~/.tbuk` dirs created `0o700`; config,
  extracted-text and SQLite DB files `0o600`. Knowledge-base content is
  personal data and need not be group/world-readable.
